*REQUERIMIENTO1  Simular las base de datos: listas blancas  listas negras  

       Especificacion1-4 Generar estructuras consistentes en memoria, que se asemejen a las bases de datos reales , los reportes de listas negras  incluirán: #IMEI , Fecha, Operador y el motivo del Reporte.
            
            Escenario1 Dadas las bases de datos de regulación nacionales que están en constante comunicación con los operadores y organismos internacionales.
                       Cuando el operador requiere hacer una mejora al sistema no puede aspirar a una copia fácilmente
                       Entonces  es necesario crear datos de prueba con la misma estructura y así generar reportes con todos los campos.  


*REQUERIMIENTO2   Mostrar un reporte de integridad por IMEI:
        
        Especificacion5: Se quiere validar si un IMEI es valido y total mente libre para salir al mercado, es primordial que el Imei que no pertenezca a las listas:  bancas y negras.  

              Escenario2 : Dadas las condiciones de validación, el IMEI tiene que tener 15 dígitos seguidos, no puede tener espacios, letras ni caracteres especiales.
                           Cuando se ingrese el IMEI a la aplicación por el usuario interesado
                           Entonces se espera que la aplicación retorne error si existe no conformidades en la estructura del IMEI

              Escenario3 : Dadas las bases de datos positivas, listas blancas, se quiere saber si el iImei consultado pertenece o no pertenece a las listas blancas
                           Cuando se ingrese el IMEI a la aplicación por el usuario interesado
                           Entonces  se espera que la aplicación retorne error si ya existe en listas blancas    
              
              Escenario4 : Dadas las bases de datos negativas, listas negras, se quiere saber si el Imei consultado tiene reporte en las listas negras
                           Cuando se ingrese el IMEI a la aplicación por el usuario interesado
                           Entonces  se espera que la aplicación retorne error y el reporte de listas negras si lo hay
              
              Escenario5 : Dados los diferentes reportes que existen de los IMEIs, seria de gran utilidad un reporte que verificara en su totalidad los registros del Imei en Bases de datos antes de salir al mercado, este reporte se efectuaría 
                           Cuando se ingrese el IMEI a la aplicación por el usuario interesado 
                           Entonces  se espera que la aplicación retorne el Reporte de integridad Mostrando que el IMEI es valido y que no esta registrado en BD positiva ni negativa
